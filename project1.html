Introduction

The world is largely visual. Television is a part of everyone’s lives. Computers and iPads are visual media. Statistically, communication is mainly visual. Research indicates that majority of communication happens visually and verbal expression remains limited. Today’s world must be made accessible for everyone, regardless of a person’s abilities or impairments. “Accessibility ensures people with disabilities can access the same information from a system as everyone else, and also gain the same benefits”.

The proposed project extends this idea of accessibility to physical interfaces, which will enable the visually challenged in various professional ways, to express their thoughts and ideas. Using Raspberry Pi, a simple yet efficient interface is developed which helps the visually challenged to convert an image of printed running text using Optical Character Recognition technology into speech. A Speech-to-Text feature is also embedded into this interface, which will convert a person’s speech into text and display the same onto a monitor.

Proposed System

I analyzed eight research papers to perform literature review about the existing solutions to the problem statement. Over the years, many papers have talked about developing an interface for the visually impaired, analyzed different Text-to-Speech and Speech-to-Text methodologies.

Finally, I broke down my proposed solution into two modules:

  1. Input of text document in English: Raspberry Pi camera module is connected to the Raspberry Pi 3B model. A text document, an A4 size sheet is placed, within the distance of 10-15 cm in front of the camera module. The camera clicks an image of this document and sends it to the OCR engine.
  2. Optical Character Recognition: Tesseract OCR engine converts the image of the text into machine-encoded language (ASCII values). Text lines are broken down according to the spacing between characters. Each word is recognized and is considered as training data in the first iteration. In the second iteration, certain words not recognized in the first iteration are recognized again.
  3. Speech Conversion: The recognized characters are displayed onto the screen. Text-to-Speech algorithm converts the text into human like voice speech in English language. Headphones are connected in the audio jack of Raspberry Pi and speech can be heard through them.
  1. English speech Input: Speech input can be obtained from USB microphone, Bluetooth headset of prerecorded audio. The input is sent to IBM Watson for text conversion.
  2. Speech-to-Text conversion by IBM Watson: IBM Watson provides with an API key and URL values for conversion of audio into text. Thus, an API key, format of the audio, path of the audio stored in the local system and the URL value is sent for conversion to text.
  3. Text output: Text received from IBM Watson is printed on the monitor attached to the Raspberry Pi.

Final Design and Solution

After analyzing all the existing models, perfoming a literature review and designing various diagrams, I finally developed a physical, working Human-Computer Interface for the visually challenged is developed with a Raspberry Pi. The interface model successfully clicks an image of the text document, recognizes the characters by Optical Character Recognition and finally converts printed running English text into English speech. This project focuses only on the textual regions; complex backgrounds are beyond the scope of it. This interface also recognizes an English speech, IBM Watson converts the same into English text and this text is displayed on the screen.

Testing

I tested my prototype with various font styles to analyze the accuracy of Text-to-Speech feature and various accents to analyze the accuracy of Speech-to-Text feature. The results are summarized in the table below. In depth testing including all the speech input and output recordings can be accessed in the final deck presentation.

Summarization of Text-to-Speech Test Cases

Font and Size Number of Words OCR Accuracy Remarks
Times New Roman - 14 110 99% Successful conversion. No errors in speech.
Courier New - 14 110 97% Very few spelling mistakes that do not affect the speech much.
Pacifico - 14 110 77% Confusing text, many spelling mistakes which results in incorrect speech.
Oswald - 14 110 88% Few spelling mistakes, only those words are affected in the speech.
Caveat - 14 110 78% Confusing text, lots of unnecessary blank spaces even though the input doesn’t have those many spaces.

Summarization of Speech-to-Text Test Cases

Accent Number of Words Accuracy Remarks
Indian 23 80% No change in text
America 23 95% No change in text
Australian 23 86% One spelling mistake
Italian 23 66% Many spelling mistakes, confusing text
Chinese 23 83% Two-three spelling mistakes, yet the context of the text is not harmed

Future Enhancements

  1. Making the model intelligent enough to recognize handwritten text
  2. Increasing the accuracy of OCR in cursive font
  3. Increasing the accuracy of Speech-to-Text in various accents
  4. Increasing the efficiency of the model by using vernacular languages
  5. Developing an offline Text-to-Speech and Speech-to-Text modules without having to overload Raspberry Pi
  6. Making the physical model compact enough to be used commercially
